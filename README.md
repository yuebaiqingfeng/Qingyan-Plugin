# 清言群管插件 - 智能群聊禁言管控系统
## 插件简介
这是一款基于大语言模型（LLM）打造的智能群聊禁言管控插件，专为社群精细化运营设计，同时提供**AI智能自动禁言**与**管理员手动指令禁言**双模式管控能力。全场景交互回复均由大模型自主生成，无固定话术模板，可完美贴合运营人设，为社群提供高效、灵活、智能的发言秩序管理方案。

## 核心功能特性
1.  **LLM驱动智能违规判定**
    基于大语言模型深度解析聊天内容，智能识别违规行为、判定违规等级，自动触发禁言操作，实现无人值守的社群秩序管控。
2.  **全链路精细化权限管控**
    - 超级管理员白名单：配置的超级管理员全程豁免禁言，不受任何管控规则限制
    - 授权管理员体系：仅名单内的授权管理员可执行手动禁言指令，严控操作权限
    - 群组白名单：可限制仅指定群组启用禁言功能，适配多群差异化运营需求
3.  **灵活可配的禁言时长管理**
    支持自定义禁言时长上下限，可设置最短/最长禁言时长（单位：秒），系统自动修正超出合规范围的时长参数，杜绝异常操作。
4.  **无模板自然拟人化回复**
    违规提醒、操作反馈、禁言通知等全场景回复内容，均由大模型自主生成，无固定话术模板，高度贴合预设人设，告别生硬的机械回复。
5.  **便捷的手动禁言指令**
    提供极简的手动禁言指令，支持精准指定用户、禁言时长、违规理由，方便管理员快速处置突发违规行为。
6.  **完善的参数校验与安全防护**
    全流程输入参数合法性校验，自动拦截格式错误、越权操作、异常时长等违规请求，避免误操作引发的运营风险。

## 手动禁言指令手册
### 指令格式
核心指令：`/qy`
完整用法：`/qy <目标用户> <禁言时长(秒)> [违规理由]`
- 必选参数：`<目标用户>` 支持用户名、@用户 两种格式
- 必选参数：`<禁言时长(秒)>` 单位为秒的正整数
- 可选参数：`[违规理由]` 禁言原因说明，支持多字文本

### 使用示例
| 指令内容 | 执行效果 |
| :--- | :--- |
| `/qy 用户名 300` | 禁言指定用户300秒（5分钟） |
| `/qy 张三 600 刷屏` | 禁言用户张三600秒（10分钟），违规理由为刷屏 |
| `/qy @某人 1800 违规内容` | 禁言@的目标用户1800秒（30分钟），违规理由为违规内容 |

## 配置项说明
### 一、基础功能开关配置
| 配置参数名 | 功能说明 | 取值范围 | 默认值 |
| :--- | :--- | :--- | :--- |
| `enable_qingyan_action` | 智能自动禁言功能总开关 | true/false | true（默认开启） |
| `enable_qingyan_command` | 手动禁言指令功能开关 | true/false | false（默认关闭） |

### 二、权限体系配置
| 配置参数名 | 功能说明 | 格式规范 | 配置示例 |
| :--- | :--- | :--- | :--- |
| `admin_users` | 超级管理员白名单，名单内用户**无法被禁言**，全程豁免管控 | 数组格式，元素为 `平台标识:用户ID` | `[qq:123456789]` |
| `allowed_users` | 授权管理员列表，仅名单内用户**有权限执行禁言操作** | 数组格式，元素为 `平台标识:用户ID` | `[qq:123456789]` |
| `allowed_groups` | 启用群组白名单，仅名单内的群组可使用本插件全部功能 | 数组格式，元素为 `平台标识:群组ID` | `[qq:123456789]` |

### 三、核心禁言规则配置
| 配置参数名 | 功能说明 | 单位 | 默认值 | 补充说明 |
| :--- | :--- | :--- | :--- | :--- |
| `min_duration` | 单次禁言最短时长限制 | 秒 | 60 | 低于该值的时长，系统将自动修正为最小值 |
| `max_duration` | 单次禁言最长时长限制 | 秒 | 2592000（30天） | 高于该值的时长，系统将自动修正为最大值 |
